# Giftia 详细使用文档

本文件包含 Giftia 插件的完整命令列表、核心配置项详解、提示词示例说明以及故障排查指南。

---

## 常用命令列表

以下命令通常需要**管理员权限**（除部分群组/非敏感查询外）：

| 命令 | 参数 | 描述 | 示例 |
| :--- | :--- | :--- | :--- |
| `/工具列表` | `[页码]` | 列出当前 Bot 已加载的 LLM 工具（函数调用列表） | `/工具列表 1` |
| `/工具解析` | `<工具名>` | 将特定工具转换为大模型可用的 XML 格式 | `/工具解析 search_chat_history` |
| `/读取记忆` | `<Bot名> <群/用户ID> <查询词>` | 根据 RAG 检索长期向量记忆 | `/读取记忆 流萤 1234567 暑假约定` |
| `/读取近期记忆` | `<Bot名> <群/用户ID> [数量]` | 查看该会话下的最新几条短期聊天总结 | `/读取近期记忆 流萤 1234567` |
| `/强制总结` | 无 | 手动强制触发对当前会话未总结消息的 passive 总结 | `/强制总结` |
| `/删除消息` | 无 | 引用某条消息发送此命令，将从本地缓存中彻底删除 | 引用消息后发送：`/删除消息` |
| `/删除记忆` | `<记忆UUID>` | 删除一条特定的长期记忆 | `/删除记忆 xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx` |
| `/清空记忆` | `<Bot名> <群/用户ID>` | 彻底清空某会话下该 Bot 的全部记忆 | `/清空记忆 流萤 1234567` |
| `/加满能量` | `<Bot名>` | 手动加满该 Bot 在当前会话的能量值（对应能量衰减限制） | `/加满能量 流萤` |
| `/清空媒体缓存` | 无 | 清空转述缓存数据库及对应的多媒体文件 | `/清空媒体缓存` |
| `/定时任务列表`| `[页码]` | 列出所有由大模型通过工具创建的定时提醒任务 | `/定时任务列表 1` |
| `/获取定时任务`| `<前缀>` | 按 Bot 或群 ID 前缀过滤定时任务 | `/获取定时任务 流萤:1234` |
| `/删除定时任务`| `<任务ID>` | 取消并删除对应的定时提醒任务 | `/删除定时任务 job_xxxxx` |
| `/读取媒体转述`| 无 | 引用图片/语音或当前消息中携带媒体时，查看对其识别转述的内容 | `/读取媒体转述`（别名 `/媒体转述`）|
| `/删除数据表` | `<表名>` | 从本地数据库中删除特定的表（危险操作） | `/删除数据表 t_xxxx` |

## 核心配置详解

你可以在后台插件配置（或 `_conf_schema.json` 自动生成的配置界面）中修改以下自定义参数。

### 1. 机器人模板列表 (`bot_template`)
可以为同一个 AstrBot 实例添加多个**机器人设定**。每个机器人的记忆和画像是物理隔离的。
- **启用机器人 (`enabled`)**: 是否启用该机器人实例。
- **机器人名称 (`name`)**: 机器人名称，如 `流萤`。名称必须唯一，用于索引机器人配置（支持中文）。
- **机器人昵称 (`nickname`)**: 机器人昵称，用于聊天历史和消息记录中的识别。
- **适配器ID列表 (`adapter_ids`)**: 绑定的平台/账号 ID（通过 `/sid` 查看），支持多平台共用同一记忆。

### 2. 小模型接话决策 (`decision_conf`)
- **启用小模型判断功能 (`enabled`)**: 推荐开启。开启后会先经过高性价比的小模型对上下文进行是否回复的决策。
- **小模型提供商 (`provider_ids`)**: 建议选择低延迟、低单价的大模型（支持配置多个以自动降级 fallback）。
- **接话分析窗口消息数 (`reply_active_window`)**: 机器人发言后，后续多少条群消息会触发分析。设置为 `0` 表示只在被 `@` 时才回复。
- **主动接话触发概率 (`proactive_probability`)**: 当分析计数为 0 且没有被 `@` 时，群友发来的每条消息按设定概率（0-100%）随机触发接话决策。
- **启用关键词监听模式 (`keyword_trigger_enabled`)**: 开启后，当群聊消息中提到设置的兴趣关键词时触发接话决策。
- **兴趣关键词列表 (`keyword_rules`)**: 支持直接输入关键词，或以“关键词:概率”格式（例如 `原神:50`）指定独立触发概率。
- **小模型判断系统提示词 (`decision_prompt`)**: 控制小模型判断逻辑的系统提示词（模版见 [流萤-小模型判断.md](prompt_example/character/流萤-小模型判断.md)）。

### 3. 大模型回复配置 (`llm_reply_conf`)
- **启用大模型回复功能 (`enabled`)**: 是否启用大模型回复。
- **大模型回复提供商 (`provider_ids`)**: 你的核心回复大模型，支持配置多个以自动降级或随机轮询。
- **多提供商选择模式 (`provider_selection_mode`)**: 支持 `fallback`（按顺序降级尝试）或 `random`（随机打乱顺序使用）。
- **大模型回复系统提示词 (`llm_reply_prompt`)**: 核心人设提示词（模版见 [天童爱丽丝-大模型回复.md](prompt_example/character/天童爱丽丝-大模型回复.md) 或 [流萤-大模型回复.md](prompt_example/character/流萤-大模型回复.md)）。

### 4. 媒体转述配置 (`caption_config`)
- **启用图片转述 (`image_caption_enabled`)** / **启用音频转述 (`audio_caption_enabled`)**: 是否开启对多媒体消息的自动多模态识别与文字翻译。
- **图片/音频转述提供商 (`image_caption_provider_ids` / `audio_caption_provider_ids`)**: 用于执行多模态转述的大模型通道。
- **按需延迟媒体转述 (`defer_caption_enabled`)**: 开启后，仅在确定需要回复时，才会调用大模型补全本次会话涉及的图片/语音转述，大幅降低盲目识别带来的 Token 开销。

### 5. 记忆检索配置 (`memory_config`)
- **启用嵌入模型 (`embedding_conf.enabled`)**: 是否启用本地或外部 Embedding 提供商，开启后才能使用长期记忆检索功能。
- **启用重排模型 (`rerank_conf.enabled`)**: 是否启用 Rerank 提供商对检索到的向量记忆进行重排，以获取更精确的参考。
- **启用被动状态维护 (`passive_memory_enabled`)**: 开启后台闲聊自动总结归纳。机器人会在后台对聊天片段进行总结，动态更新用户画像、群画像、好感度与关系头衔。
- **被动总结模型提供商 (`passive_memory_provider_ids`)**: 指定用于后台总结与画像提取的大模型通道。
- **被动总结静默触发阈值 (`passive_memory_silence_threshold`)**: 机器人发言后，群聊沉寂静默多少条消息后触发后台被动总结。
- **被动总结溢出触发阈值 (`passive_memory_overflow_threshold`)**: 机器人活跃时，未处理缓存消息累计达到多少条后触发后台总结。

## 故障排查与说明

- **数据库锁定/报错**：本插件底层使用 `LanceDB` 存储向量记忆与 SQLite。请确保插件数据目录下文件可读写。若数据库出现异常损坏，可尝试通过命令删除特定的表或手动清空缓存。
- **接话过于频繁**：请调低小模型的 `主动接话触发概率`，调小 `接话分析窗口消息数`，并合理设置 `并发配置` 下的 `群组节流时间` 和 `用户防抖`。
